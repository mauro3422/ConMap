# v35.0 — "Snake Layout Engine"

*   **Fecha:** 22 de Abril, 2026
*   **Estado:** Producción / Final.

## 🚀 Nuevas Funcionalidades
* **Snake Layout (Viborita 3x3)**: Implementación de un flujo de cuadrícula (`grid-flow`) automatizado para ramas muy largas (ej: Históricas), permitiendo organizar la lectura de nodos en formato de "párrafo" (3x3) para maximizar la legibilidad de la pantalla y aprovechar el espacio vacío a la izquierda del root.
* **Algoritmo de Renderizado Bidireccional**: Mejoras en `renderer.js` para trazar Curvas Bézier dinámicamente tanto de Izquierda a Derecha como de Derecha a Izquierda, evitando el cruce visual de las flechas.
* **Offset Inicial de Rama**: El motor de posicionamiento envía estratégicamente a la izquierda el punto de nacimiento del `S-Shape` para asegurar que el flujo occidental (Izquierda a Derecha) se respete en el primer renglón, sin chocar contra las demás ramas estructurales.

## 🐛 Corrección de Errores (Bugfixes)
* **Hitbox Collision Fix**: Aislamiento del algoritmo de viborita estrictamente a nodos de la temática correspondiente (`themeName === 'gold'`). Esto evita que nodos aislados terminales (como "Conclusiones" de ramas verticales) caigan erróneamente en el comportamiento horizontal y rompan el diseño.
* **Propagación Matemática Optimizada (Zero-Gap)**: Refactor del cálculo topológico `subtreeHeight` y `subtreeWidth` en las ramas `S-Shape`. Se eliminó la acumulación infinita de espacios vacíos calculando inteligentemente las "filas reales" del contenedor. Esto erradicó el "vacío vertical muerto" entre la rama temporal superior y la primera rama vertical inferior.
